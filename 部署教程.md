# 🚀 Fizz Clone 部署教程

## 这是什么技术栈？
- **Next.js 14** — 前端框架（React）
- **Supabase** — 真实数据库 + 用户认证 + 实时更新（免费）
- **Vercel** — 自动部署，全球 CDN（免费）

部署完成后：
✅ 真实用户注册/登录
✅ 多人共享同一数据库
✅ 在线人数是真实统计的
✅ 消息实时推送
✅ 全球可访问的域名

---

## 第一步：建立 Supabase 数据库（免费）

1. 打开 [https://supabase.com](https://supabase.com) → 点击 **Start your project**
2. 用 GitHub 账号登录
3. 点击 **New project**
4. 填写：
   - **Name**：fizz-clone
   - **Database Password**：设个强密码（记住它）
   - **Region**：选 `Northeast Asia (Tokyo)` 或 `Singapore`
5. 等待约 2 分钟项目创建完成

### 建表

6. 进入项目后，点击左侧 **SQL Editor**
7. 点击 **New query**
8. 把 `supabase-schema.sql` 文件里的全部内容粘贴进去
9. 点击右上角 **Run**（或按 Ctrl+Enter）
10. 看到 "Success" 说明建表成功

### 获取 API 密钥

11. 点击左侧 **Settings** → **API**
12. 找到：
    - **Project URL**：类似 `https://abcdefg.supabase.co`
    - **anon public key**：一长串字符

## 第二步：把代码放到 GitHub

1. 在 [https://github.com](https://github.com) 新建一个 repository，名称随意，设为 Public
2. 在本地解压我给你的项目文件夹
3. 打开终端（命令行），进入文件夹：
   ```bash
   cd fizz-app
   git init
   git add .
   git commit -m "init fizz clone"
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   git push -u origin main
   ```

---

## 第三步：在 Vercel 上部署（免费，自动上线）

1. 打开 [https://vercel.com](https://vercel.com) → 用 GitHub 账号登录
2. 点击 **New Project**
3. 找到你刚才建的 GitHub 仓库，点击 **Import**
4. 在 **Environment Variables** 里添加两个变量：

   | 变量名 | 值 |
   |--------|-----|
   | `NEXT_PUBLIC_SUPABASE_URL` | 你的 Supabase Project URL |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 你的 anon public key |

5. 点击 **Deploy**
6. 等 1~2 分钟，你会得到一个 `https://你的项目名.vercel.app` 的域名
7. **完成！** 打开这个链接就是你的 Fizz 了

---

## 第四步：关闭 Supabase 邮件验证（可选，方便测试）

默认 Supabase 注册时需要验证邮箱，你可以关闭它：

1. 进入 Supabase → **Authentication** → **Providers** → **Email**
2. 关闭 **Confirm email**
3. 保存

这样注册后可以直接登录，不需要验证邮件。

## 本地开发运行（可选）

```bash
cd fizz-app

# 把 .env.example 复制成 .env.local
cp .env.example .env.local

# 编辑 .env.local，填入你的 Supabase URL 和 Key
nano .env.local

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 打开 http://localhost:3000
```

## 常见问题

**Q: 注册后提示要验证邮箱但收不到邮件？**
A: 按上面"第四步"关闭邮件验证，或者检查垃圾邮件箱。

**Q: 数据库连接报错？**
A: 检查 `.env.local` 或 Vercel 环境变量里的 URL 和 Key 是否正确，注意不要有多余空格。

**Q: 想绑定自己的域名？**
A: 在 Vercel → 项目设置 → Domains 里添加你的域名即可。
